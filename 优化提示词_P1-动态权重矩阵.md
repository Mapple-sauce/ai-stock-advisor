# 优化方案 1/8：动态权重矩阵

## 优先级
🥇 最高优先级

## 当前问题
`scanner/screener.py` 中的 `WEIGHTS_LOW` 和 `WEIGHTS_MOMENTUM` 是固定值。
无论市场风格怎么变（价值市/成长市/震荡市/单边市），都用同一套权重。
因子有效性在 A 股衰减很快（通常 1-3 年），固定权重会逐渐失效。

## 目标
让系统能定期（每季度或每月）根据最近一段时间的回测表现，自动调整因子权重。
权重不再写在代码里硬编码，而是从回测结果动态生成。

## 涉及文件
- `scanner/screener.py` — 修改权重定义方式
- `backtest/optimizer.py` — 新增权重自动生成函数
- `config.py` — 可选：增加权重更新周期配置

## 实现思路

### 1. 新增 `backtest/optimizer.py` 中的函数

```python
def find_optimal_weights(
    symbols: list[str] = TEST_POOL,
    mode: str = "low_position",
    lookback_months: int = 6,   # 使用最近6个月的数据
    hold_days: int = 20,
    iterations: int = 20000,    # 每次随机搜索2万组
) -> dict:
    """
    自动寻优：在最近N个月的数据上找到最优权重组合

    输出: 与 WEIGHTS_LOW 完全兼容的权重字典
    """
```

算法：
1. 从 `TEST_DATES` 中筛选出最近 `lookback_months` 个月的日期
2. 对每个日期→每只股票跑 `_score_fast()` 评分
3. 随机生成 20000 组权重配置（Dirichlet 分布保证和为1）
4. 用 `score_config()` 评分函数选出最优权重
5. 返回最优权重 + 准确率 + 多空差

权重加载策略：

- 如果有且未过期（比如30天内），直接用

### 3. 新增 `python main.py optimize` 命令

触发流程：

4. 打印优化前后的准确率对比

## 验证方法

1. `python main.py optimize` → 应该输出优化后的权重和准确率
2. 对比新旧权重在同批数据上的准确率：
   - 旧权重低位模式 ~61.5%
   - 新权重应该 >= 旧权重（因为是在同样的数据上寻优的）
3. `python main.py backtest 2026-01-15` → 用新权重跑一次回测
4. 检查 `backtest/engine.py` 中是否引用了动态权重

## 风险提示
- 在太少数据上寻优容易过拟合，至少用 6 个月以上的数据
- 建议优化结果 vs 基线用不同的时间窗口验证（trainset vs testset 分离）
